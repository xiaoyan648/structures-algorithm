
### 一、稳定排序算法（6种常见）
#### 1. 冒泡排序（Bubble Sort）
- 原理：相邻元素两两比较，不满足顺序则交换，相等元素不交换。  
- 稳定性：稳定（相等元素无交换，相对顺序不变）。

#### 2. 插入排序（Insertion Sort）
- 原理：将元素逐个插入已排序区间的合适位置，插入时不跨越相等元素。  
- 稳定性：稳定（相等元素会插入到前一个相等元素的后面，保留原顺序）。

#### 3. 归并排序（Merge Sort）
- 原理：分治思想，将数组拆分后合并，合并时按“前半部分优先”原则处理相等元素。  
- 稳定性：稳定（合并阶段，若两元素相等，先取左半部分的元素，保留原始相对顺序）。

#### 4. 计数排序（Counting Sort）
- 原理：基于“元素值为索引”统计频次，再按顺序输出元素，相等元素按统计顺序排列。  
- 稳定性：稳定（需按原序列顺序处理相等元素，避免覆盖相对位置）。

#### 5. 桶排序（Bucket Sort）
- 原理：将元素分配到不同桶中，桶内单独排序（通常用稳定排序如插入排序），最后合并桶。  
- 稳定性：稳定（桶内排序用稳定算法，桶间按顺序合并，相等元素保留原顺序）。

#### 6. 基数排序（Radix Sort）
- 原理：按“低位到高位”（或反之）逐位排序，每一位排序用稳定算法（如计数排序）。  
- 稳定性：稳定（每一位排序的稳定性保证了整体相等元素的相对顺序）。


### 二、不稳定排序算法（4种常见）
#### 1. 选择排序（Selection Sort）
- 原理：每次找未排序区间的最小值，与未排序区间的第一个元素交换。  
- 不稳定性原因：交换可能跨越相等元素。例如序列 `[2, 2, 1]`，第一次会把末尾的1和第一个2交换，导致两个2的相对顺序颠倒（变成 `[1, 2, 2]`，但原第二个2跑到了第一个2前面）。

#### 2. 快速排序（Quick Sort）
- 原理：选基准元素，将小于/大于基准的元素分到两侧，递归排序。  
- 不稳定性原因：基准元素与其他元素交换时，可能打乱相等元素顺序。例如序列 `[3, 2, 2, 1]`，若基准选3，交换后3到末尾，可能导致两个2的相对位置变化。

#### 3. 希尔排序（Shell Sort）
- 原理：按“增量”分组，每组用插入排序，逐步缩小增量至1。  
- 不稳定性原因：分组排序时，相等元素可能被分到不同组，跨组插入交换会破坏相对顺序。

#### 4. 堆排序（Heap Sort）
- 原理：构建大顶堆/小顶堆，每次提取堆顶元素（最大值/最小值），重新调整堆。  
- 不稳定性原因：调整堆时，元素交换可能跨越相等元素。例如堆中相等元素的父节点与子节点交换，会打乱原相对顺序。


### 三、关键总结
| 类别       | 常见算法                                  | 核心特点（稳定性相关）       |
|------------|-------------------------------------------|------------------------------|
| 稳定排序   | 冒泡、插入、归并、计数、桶、基数          | 保留相等元素原始相对顺序     |
| 不稳定排序 | 选择、快速、希尔、堆                      | 可能打乱相等元素相对顺序     |
